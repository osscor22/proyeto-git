6. Ejecución con JUnit (La Clase Runner)

Para agrupar, orquestar y ejecutar los archivos .feature y sus respectivos Step Definitions, utilizamos JUnit. Para que JUnit entienda que debe ejecutar casos de Cucumber, creamos una clase especial conocida como el Runner.

¿Cómo funciona el Runner?

El Runner es una clase Java vacía que utiliza anotaciones específicas en su cabecera para configurar el entorno de ejecución:

  • @RunWith(Cucumber.class): Le indica a JUnit que debe usar el motor de Cucumber para ejecutar esta clase.
  • @CucumberOptions: Permite pasar opciones de configuración como la ubicación de los archivos feature, la ubicación de los step definitions (glue), los tags a ejecutar y los plugins para reportes.
🏃 TestRunner.java 
import org.junit.runner.RunWith;
import io.cucumber.junit.Cucumber;
import io.cucumber.junit.CucumberOptions;

@RunWith(Cucumber.class)
@CucumberOptions(
    // Ruta directa a los archivos feature
    features = "src/test/resources/features",
    
    // Paquete donde están los Step Definitions y Hooks
    glue = "steps",
    
    // Filtro para ejecutar solo pruebas con este tag (opcional)
    tags = "@Regresion",
    
    // Generación de un reporte HTML básico
    plugin = {"pretty", "html:target/cucumber-reports/report.html"},
    
    // Validar si están declarados los steps sin ejecutarlos
    dryRun = false,
    
    // Mejora la legibilidad de la salida en consola
    monochrome = true
)
public class TestRunner {
    // La clase permanece vacía
}

¿Cómo ejecutarlo? Puedes dar clic derecho sobre TestRunner.java en tu IDE y seleccionar "Run 'TestRunner'", o usar la consola de Maven: mvn clean test.
