8. Cross-Browser Testing (Múltipes Navegadores)

Para probar en Chrome, Firefox, Edge, etc., necesitamos abstraer la inicialización del WebDriver ("ejecutar en otros navegadores"). Ya no usamos directamente new ChromeDriver() en nuestras clases, sino que usamos una clase de fábrica (Factory) o un Hook centralizado.

Implementando el Soporte Multibrowser

Puedes leer una variable de entorno para saber qué navegador levantar. Así, se ejecuta globalmente según un parámetro enviado en consola:

🔧 BrowserSetup.java 
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.firefox.FirefoxDriver;
import org.openqa.selenium.edge.EdgeDriver;

public class BrowserSetup {

    public static WebDriver getDriver() {
        // Leer variable desde Maven (e.g. mvn test -Dbrowser=firefox)
        String browser = System.getProperty("browser", "chrome").toLowerCase();
        
        WebDriver driver;

        switch (browser) {
            case "firefox":
                driver = new FirefoxDriver(); // Selenium Manager se encarga del gecko driver
                break;
            case "edge":
                driver = new EdgeDriver();
                break;
            case "chrome":
            default:
                driver = new ChromeDriver();
                break;
        }
        
        driver.manage().window().maximize();
        return driver;
    }
}

¿Cómo ejecutar desde consola?

  • Ejecutar Chrome por defecto: mvn clean test
  • Ejecutar en Firefox: mvn clean test -Dbrowser=firefox
  • Ejecutar en Edge: mvn clean test -Dbrowser=edge
