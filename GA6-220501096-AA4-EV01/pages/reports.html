7. Generación de Reportes

La visibilidad de los resultados es clave. Cucumber y diversas herramientas ofrecen formas nativas o de terceros para generar reportes visuales después de la ejecución ("cómo sacar reportes").

Reporte Nativo de Cucumber (HTML y JSON)

Al configurar el Runner, puedes indicar plugins para generar reportes básicos. Es la técnica más rápida:

🏃 Configurar el Runner 
@CucumberOptions(
    features = "src/test/resources/features",
    glue = "steps",
    plugin = {
        "pretty", 
        "html:target/cucumber-reports/report.html",
        "json:target/cucumber-reports/Cucumber.json"
    }
)

Después de ejecutar mvn test con JUnit, ve a la carpeta target/cucumber-reports/ y abre report.html en tu navegador.

Reportes Avanzados: Serenity BDD o Allure

Si buscas reportes empresariales con pantallazos y estadísticas detalladas, puedes integrar librerías como Allure Reports o usar Serenity BDD. Con Serenity por ejemplo, al ejecutar mvn clean verify serenity:aggregate obtienes en target/site/serenity/index.html un "Living Documentation" muy completo, donde las pruebas fallidas automáticamente incluyen una captura de pantalla.

💡 Tip Pro: Configura Maven Surefire Plugin en tu pom.xml para generar la carpeta /target de forma consistente cada vez que corras las pruebas desde consola.
